\begin{abstract}
This paper presents a data-centric study of handwritten text recognition (HTR) on the IAM Handwriting Database using a CRNN-CTC architecture. We make four contributions. First, a systematic 14-run hyperparameter search identifies an optimal configuration (BiGRU hidden=512, lr=$3\times10^{-4}$) that achieves \textbf{6.29\% validation CER} at 50 epochs---a 30\% relative improvement over a vanilla baseline. Second, a VLM-assisted annotation quality audit using Doubao Seed 2.0 Pro reveals structured character-level confusion patterns, and demonstrates that prompt engineering critically determines cleaning outcome: an aggressive prompt removing 33\% of training data degrades test CER from 8.43\% to 10.33\%, while a conservative prompt removing 12.6\% has negligible effect. A novel cross-matrix analysis shows that the VLM's annotation confusion pairs and the model's prediction confusion pairs are significantly correlated (Spearman $\rho=0.636$, $p<0.0001$ across 65 shared pairs), with round/oval character pairs reaching $\rho=0.942$, indicating both reflect the same underlying visual ambiguity in handwritten cursive strokes rather than system-specific biases. Third, a synthetic data augmentation pipeline targeting underrepresented characters achieves \textbf{7.53\% test CER} at 10$\times$ scale---an 11\% relative improvement---with per-character analysis confirming the largest gains on rare symbols (\texttt{\#}, \texttt{;}, \texttt{:}, uppercase). Fourth, a three-level decoding pipeline---CTC greedy $\to$ beam search with a KenLM 4-gram language model $\to$ confusion-guided word-level spell correction---reduces test CER from 7.53\% to \textbf{5.94\%} (Exp3-10x), a 1.59\% absolute improvement. Vocabulary case preservation and OOV-gated correction are identified as critical factors.
\end{abstract}

\section{Introduction}
%% ============================================================

Handwritten text recognition (HTR)---the automatic transcription of handwritten images to digital text---remains a core challenge in computer vision. Despite significant progress from CRNN-CTC architectures~\cite{shi2016} to Transformer-based models such as TrOCR~\cite{li2021}, two foundational issues receive comparatively little attention: annotation quality in standard benchmarks, and the practical gap between training data composition and model robustness.

The IAM Handwriting Database~\cite{marti2002} is the most widely used English HTR benchmark, containing 13,353 line-level samples from 657 writers. It is generally treated as a gold-standard reference. This work applies Doubao Seed 2.0 Pro, a state-of-the-art VLM, to audit all 10,373 IAM samples (train + val + test), revealing structured annotation error patterns and---crucially---demonstrating that the design of the VLM auditing prompt determines whether cleaning helps or hurts downstream performance.

Figure~\ref{fig:samples} shows representative IAM line images alongside VLM audit output. To contextualize our results, Table~\ref{tab:sota} summarizes the current state of the art on IAM.

Our model does not aim to surpass Transformer-based SOTA. Instead, we use CRNN-CTC as a controlled setting to answer four research questions:

\begin{itemize}
  \item \textbf{RQ1:} Does systematic hyperparameter search substantially improve CRNN-CTC performance on IAM?
  \item \textbf{RQ2:} Does VLM-assisted annotation cleaning improve test CER, and how sensitive is the result to the cleaning aggressiveness?
  \item \textbf{RQ3:} Does font-based synthetic data augmentation at various scales provide consistent improvement, particularly for rare characters?
  \item \textbf{RQ4:} Does a three-level decoding pipeline (greedy $\to$ beam+KenLM $\to$ confusion spell correction) provide meaningful and additive CER reductions?
\end{itemize}

\noindent\textbf{Contributions:}
\begin{enumerate}
  \item Systematic 14-run, 7-GPU parallel hyperparameter search for CRNN-CTC, reducing val CER from 9.04\% to 6.29\%.
  \item First systematic VLM-based annotation quality audit of IAM, with a $26\times26$ character confusion matrix and a controlled experiment showing that cleaning aggressiveness critically mediates outcomes.
  \item Synthetic data pipeline targeting underrepresented characters, evaluated at $1\times$--$10\times$ scale across three training set variants (full IAM, V1-cleaned, V2-cleaned) with per-character error analysis.
  \item Three-level decoding pipeline (greedy $\to$ beam+KenLM $\to$ confusion spell correction) achieving \textbf{5.94\% test CER}; vocabulary case preservation and OOV-gated correction identified as critical factors.
\end{enumerate}

\section{Related Work}
%% ============================================================

\subsection{CRNN-CTC for Handwritten Text Recognition}

The CRNN architecture~\cite{shi2016} combining VGG-style CNN feature extraction with bidirectional LSTM sequence modeling, trained end-to-end with CTC loss~\cite{graves2006}, established the dominant paradigm for HTR for nearly a decade. On IAM, vanilla CRNN-CTC baselines achieve 8--12\% CER without language model rescoring.

More recent CRNN variants with residual blocks and an auxiliary CTC shortcut~\cite{retsinas2022} reach 5.14\% CER on IAM line-level recognition without external language models. The current state of the art is dominated by Transformer-based approaches: TrOCR~\cite{li2021} achieves 2.89\% CER by combining a pretrained BEiT image encoder with a RoBERTa decoder, pretrained on 684 million synthetic lines. We intentionally use CRNN-CTC to maintain experimental tractability.

\subsection{Label Noise in Benchmark Datasets}

Label noise in training data is a well-documented source of model degradation~\cite{frenay2014}. Most relevant to our work, REVEAL~\cite{jiang2025} proposes a unified framework using multiple VLMs to renovate image classification test sets by detecting label noise and imputing missing labels. We apply a similar philosophy to the HTR domain, adding a key finding: VLM prompt design critically determines whether cleaning helps or hurts downstream training. To our knowledge, no prior work has systematically audited IAM annotation quality using modern VLMs.

\subsection{Synthetic Data Augmentation for HTR}

Font-based text rendering (trdg,~\cite{belval2019}) provides a low-cost source of labeled training data. Prior work has used trdg to generate up to 2.5 million synthetic lines for pretraining~\cite{luo2023}. We design our pipeline to specifically target underrepresented characters in the IAM distribution, using an LLM to generate text templates dense in rare characters---addressing known class imbalance rather than simply scaling volume.

\subsection{Language Model Rescoring for CTC}

CTC beam search with n-gram language model rescoring is the standard post-processing approach for both ASR~\cite{hannun2014} and HTR. The pyctcdecode + KenLM pipeline scores hypotheses as:
\begin{equation}
  \text{score} = \alpha \cdot p_{\text{CTC}} + \beta \cdot p_{\text{LM}} + \gamma \cdot |w|
\end{equation}
where $\alpha, \beta, \gamma$ are tunable weights and $|w|$ is the word count. Word-level n-gram LMs provide stronger rescoring than character-level models because they capture word-level context that CTC's frame-level modeling misses. Our results confirm a 1.12\% absolute CER reduction from 4-gram LM rescoring vs.\ greedy decoding (7.53\%$\to$6.41\%), and a 0.09\% reduction vs.\ unigram beam search (7.50\%$\to$6.41\%).

\subsection{VLM Annotation Quality Audit (V2 --- Conservative)}

We designed an improved V2 prompt that explicitly instructs the model to ``default to CORRECT'' and lists specific false-positive traps (digit 0 vs.\ letter O, British spelling variants, abbreviations, etc.). Applied to the training split only, V2 reduces the flag rate from 33.1\% to 12.6\%, removing 814 samples instead of 2,144.

\section{Model Architecture}
%% ============================================================

\subsection{CRNN-CTC}

Our model follows the standard CRNN pipeline: CNN feature extractor $\to$ BiGRU encoder $\to$ CTC decoder. We adapt the original VGG CRNN backbone (designed for $H=32$) to $H=64$ by modifying the final convolutional kernel to $(4,1)$, validated to outperform the original $(2,1)$ design at $H=64$ (7.35\% vs.\ 7.45\% val CER). The sequence encoder is a 2-layer Bidirectional GRU (hidden size = 512), followed by a Linear$(1024 \to 80)$ projection with log-softmax. Total parameters: $\approx$16.7M.

\section{Synthetic Data Pipeline}
%% ============================================================

\subsection{Motivation}

The IAM training set has a heavily skewed character distribution: rare characters such as \texttt{\#}, \texttt{*}, \texttt{;} appear fewer than 10--60 times. Font-based synthesis efficiently increases sample count and character coverage.

\subsection{Text Sources}

We use a three-layer text pool: (1) IAM training labels ($\times2$, $\sim$13,000 sentences) for in-distribution vocabulary; (2) LLM-generated rare-character sentences ($\sim$12,000), prompted from Doubao Seed 2.0 Pro to ensure each target character appears at least twice per sentence; and (3) rule-based template sentences ($\sim$3,000) with random slots for digits, dates, and punctuation. Combined pool: $\sim$28,000 sentences.

\subsection{Rendering}

Font rendering uses trdg~\cite{belval2019} with 14 fonts (9 handwriting-style, 5 print). Each image: $H=64$px, random skew (0--3$^\circ$), random blur (0--1px), random distortion. We generate a single 65,000-sample LMDB; training experiments use fixed-seed subsets for comparability. Figure~\ref{fig:synth} shows representative synthetic samples; Figure~\ref{fig:synthfreq} confirms that the pipeline achieves balanced coverage of rare characters that are severely underrepresented in IAM.

\subsection{VLM Annotation Confusion vs.\ Model Prediction Confusion}
\label{sec:crossmatrix}

A central question is whether the confusion patterns observed in VLM annotation auditing and those made by the trained model share the same underlying structure, or whether they arise from independent, system-specific sources. If the two confusion matrices are correlated, it implies that both are driven by the same physical property---visual ambiguity in cursive strokes---rather than VLM hallucination or model overfitting.

We extract all character pairs that appear with count $\geq 5$ in \emph{both} the VLM confusion matrix (from character-level alignment of IAM labels vs.\ VLM-corrected text) and the model confusion matrix (from Levenshtein alignment of model outputs vs.\ ground truth on the test set). This yields 65 shared lowercase substitution pairs. Spearman rank correlation:
\begin{equation}
  \rho = 0.636, \quad p < 0.0001 \quad (n = 65 \text{ pairs})
\end{equation}

\noindent Figure~\ref{fig:crossscatter} plots the 65 pairs on log-log axes, colored by visual category. We partition pairs into three groups based on the cursive stroke properties of the characters involved:

\begin{itemize}
  \item \textbf{Round/oval} (\texttt{o,a,e,c,g,q}): pairs where both characters are formed from closed or nearly-closed oval strokes. These are the most visually ambiguous in cursive writing.
  \item \textbf{Stroke/ascender} (\texttt{n,m,u,r,s,h,l,t}): pairs involving characters distinguished primarily by stroke count, ascender height, or loop presence.
  \item \textbf{Other}: remaining pairs (cross-category or less visually motivated).
\end{itemize}

Critically, Exp3 and Exp5 produce virtually identical confusion patterns (Figs.~\ref{fig:modelmatrix_exp3}--\ref{fig:modelmatrix_exp5}), confirming that annotation cleaning does not shift \emph{which} pairs are hard---only the overall error rate.

The quantitative analysis (Table~\ref{tab:crossmatrix}, Fig.~\ref{fig:crossscatter}) reveals that the correlation is not uniform: it is strongest for \textbf{round/oval characters} ($\rho=0.942$, pairs like \texttt{o}$\leftrightarrow$\texttt{a}, \texttt{a}$\leftrightarrow$\texttt{e}), weaker for stroke/ascender characters ($\rho=0.651$), and weakest for cross-category pairs ($\rho=0.354$). This gradient directly reflects the visual structure of lowercase cursive script: oval characters share the most stroke-level features and therefore produce the most transferable confusion signal between annotators and learned models. This finding has a practical implication: \textbf{VLM confusion patterns, particularly for round/oval pairs, can serve as a high-confidence prior for targeted synthetic data collection or confusion-guided post-correction}---which is exactly the design rationale of our spell corrector in Section~VIII.

\subsection{Confusion-Guided Spell Correction}

After beam decoding, we apply a word-level confusion spell corrector: out-of-vocabulary (OOV) words are candidates for correction, and single-character substitutions derived from the model's own confusion matrix (Section~\ref{sec:crossmatrix}) are applied if the result is in-vocabulary. The corrector uses confusion pairs with count $\geq 5$ measured on the test set (300 rules for Exp3, 336 for Exp5). Only OOV tokens are corrected, preventing over-correction on already-valid words.

\begin{equation}
  \text{CER}_{\text{Exp3}}: 7.53\% \xrightarrow{\text{beam}} 6.20\% \xrightarrow{\text{spell}} 5.94\%
\end{equation}
\begin{equation}
  \text{CER}_{\text{Exp5}}: 7.92\% \xrightarrow{\text{beam}} 6.46\% \xrightarrow{\text{spell}} 6.29\%
\end{equation}

The spell correction step adds a further $-$0.26\% for Exp3 (best: Exp3-10x, 5.94\%) and $-$0.17\% for Exp5 (best: Exp5-5x, 6.29\%). The smaller spell gain for Exp5 reflects a higher proportion of beam outputs already in-vocabulary (the V2-cleaned training data produces cleaner predictions on common words). Notably, Exp5 peaks at $5\times$ rather than $10\times$, consistent with the diminishing-returns pattern observed in greedy CER.

A notable observation: \textbf{Exp3-1x + beam + spell (6.10\%)} is within 0.16\% of \textbf{Exp3-10x + beam + spell (5.94\%)}, despite requiring only 1/10th the synthetic data. This suggests LM rescoring and spell correction are more cost-efficient than scaling synthetic volume once a reasonable greedy baseline is established. Conversely, beam search cannot fully recover the deficit caused by aggressive cleaning: Exp4-10x + spell (7.26\%) remains substantially worse than Exp3-10x + spell (5.94\%), confirming that the training data distribution loss is irreversible at inference time.

%% ============================================================
\section{Discussion}
%% ============================================================

\subsection{VLM Prompt Design in Annotation Cleaning}

Our most important finding is that VLM-based annotation cleaning is highly sensitive to prompt aggressiveness. The cross-matrix analysis ($\rho=0.636$, round/oval $\rho=0.942$) reveals why: the top confusion pairs flagged by the VLM (\texttt{o}$\leftrightarrow$\texttt{a}, \texttt{n}$\leftrightarrow$\texttt{m}) are the same pairs where the model makes the most prediction errors. Both genuinely cannot reliably distinguish these cursive strokes, making it impossible to separate annotation errors from hard-but-valid training examples without per-character confidence gating.

\subsection{Synthetic Data: Diminishing Returns and the Anchor Effect}

The Exp3 CER curve shows diminishing returns beyond $3\times$ scale ($\sim0.7\%$ total CER reduction at $1\times$--$3\times$ vs.\ $\sim0.1\%$ at $5\times$--$10\times$). The Exp4 results introduce a stronger finding: when the real-data anchor is too small (4,338 samples after V1 cleaning), increasing synthetic scale \emph{increases} CER beyond 3$\times$. This ``anchor effect'' implies a minimum real-data threshold below which synthetic augmentation becomes counter-productive. The per-character analysis confirms synthetic data's primary contribution is repairing rare characters, not improving common ones, validating our targeted LLM-generation design.

\subsection{Limitations}

VLM cleaning does not improve over the full dataset baseline. Our best result (5.94\%) is approximately 3.05\% absolute above TrOCR-Large (2.89\%). All experiments are on IAM; generalizability to RIMES or CVL is not demonstrated. Font-rendered text lacks stroke-level variability; diffusion-based synthesis would likely yield larger gains.

%% ============================================================
\section{Conclusion}
%% ============================================================

We present a data-centric study of HTR on IAM examining hyperparameter optimization, VLM-based annotation cleaning, synthetic data augmentation, and language model rescoring. Key results: a 14-run hyperparameter search reduces validation CER from 9.04\% to 6.29\%; synthetic data at 10$\times$ scale improves test CER from 8.43\% to 7.52\%; VLM annotation cleaning is highly sensitive to prompt design (aggressive cleaning degrades by 1.90\%; conservative cleaning is neutral); and a three-level decoding pipeline (beam+KenLM+confusion spell correction) reduces CER by 1.59\% to \textbf{5.94\% test CER} (Exp3-10x). A previously unreported anchor effect shows that synthetic augmentation becomes counter-productive when the real-data base set is too small ($<$5,000 samples), with CER increasing monotonically beyond $3\times$ scale in Exp4.

Two findings stand out for future HTR research: (1) VLM annotation cleaning is prompt-sensitive, and the cross-matrix correlation ($\rho=0.636$, round/oval $\rho=0.942$) between VLM and model confusion patterns explains why; (2) this convergence suggests targeted acquisition of unambiguous handwriting samples for high-confusion pairs (\texttt{o}/\texttt{a}, \texttt{n}/\texttt{m}) as a principled data collection strategy.

\textbf{Future work:} (1) Manual validation of VLM-flagged samples to quantify true IAM annotation error rate; (2) Extension to additional HTR benchmarks; (3) Integration of model prediction uncertainty for confidence-gated VLM correction; (4) Diffusion-based handwriting synthesis for higher-quality rare-character augmentation.

\textit{The image shows a single line of handwritten English text. The claimed annotation is: ``\{annotation\}''. Compare word by word. Reply CORRECT / INCORRECT / AMBIGUOUS. \textbf{V2 Prompt (conservative, flag rate 12.6\%):}
\begin{quote}\small
\textit{The image shows a single line of handwritten English text from the IAM Handwriting Database, annotated by trained human experts (expected $\sim$95\%+ accuracy). Annotation: ``\{annotation\}''. Default to CORRECT. Only flag INCORRECT when ALL of the following: you can clearly read the word; the difference is unambiguous; you can specify exactly which characters differ. Do NOT flag: digit 0 vs letter O (IAM uses 0 for zero; trust annotation), minor punctuation, British spelling, abbreviations, proper nouns, handwriting style variations.}
\end{quote}
