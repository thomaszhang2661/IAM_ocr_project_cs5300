## 摘要

本文基于IAM手写数据库，使用CRNN-CTC架构对手写文字识别（HTR）进行以数据为中心的系统性研究，提出四项主要贡献。

**超参数优化：** 通过14轮、7 GPU并行的系统性搜索，确定最优配置（BiGRU隐藏层=512，lr=3×10⁻⁴），在50轮训练内将验证集CER从9.04%降至**6.29%**，相对提升30%。

**VLM标注质量审计：** 使用豆包Seed 2.0 Pro对IAM全量样本进行标注审计，揭示结构化的字符级混淆规律，`o↔a`和`n↔m`为主要混淆对。关键发现：审计提示词的设计决定清洗效果——激进提示词（V1）删除33%样本导致测试CER从8.43%恶化至10.33%，保守提示词（V2）删除12.6%则影响可忽略。跨矩阵分析发现VLM标注混淆对与模型预测混淆对之间存在显著正相关（Spearman ρ=0.559，p<0.0001，160个共同对），表明两者反映的是手写草书中相同的视觉模糊性，而非各自系统的特有偏差。

**合成数据增强：** 针对欠采样字符构建合成数据流水线，10×规模下测试CER达到**7.52%**（相对基线提升11%）；逐字符分析确认最大增益集中于稀有符号（`#`、`;`、`:`、大写字母）。

**语言模型解码：** 在公开英文语料上训练KenLM 4-gram词级语言模型，beam search将CER进一步降至**6.41%**（相较贪心解码绝对提升1.12%）；发现词表保留大小写（而非全部转小写）可额外降低0.23% CER，因为IAM中大量专有名词在小写词表中被当作未知词惩罚。

## 1. 引言

手写文字识别（HTR）是计算机视觉领域的核心挑战之一。IAM手写数据库（Marti & Bunke，2002）是使用最广泛的英文HTR基准，含657位书写者的13,353条行级样本。该数据集通常被视为金标准参考——但据我们所知，尚无针对其标注质量的系统性审计发表。本文填补了这一空白，并在此过程中发现：**VLM能够识别的字符视觉混淆规律与训练完成的深度学习模型的错误规律高度吻合**，二者均反映了手写草书中相同的客观视觉模糊性。

**当前IAM测试集SOTA对比（表1）：**

| 模型 | 架构 | 测试CER | 语言模型 |
|---|---|---|---|
| DRetHTR（2025） | 仅解码器RetNet | 2.26% | 有 |
| TrOCR-Large（Li等，2021） | BEiT + RoBERTa | 2.89% | 无 |
| Retsinas等（2022） | CRNN + CTC shortcut | 5.14% | 无 |
| **本文最优（Exp3-10x + 4-gram beam）** | CRNN-CTC BiGRU | **6.41%** | 4-gram |
| **本文（Exp3-10x，贪心解码）** | CRNN-CTC BiGRU | **7.52%** | 无 |
| **本文（Exp1，完整IAM，贪心解码）** | CRNN-CTC BiGRU | **8.43%** | 无 |

*表1：IAM行级测试CER。本文以CRNN-CTC为受控设置，解耦数据层面的影响与架构改进。*

**研究问题：**
- **RQ1：** 系统性超参数搜索能否显著提升CRNN-CTC在IAM上的性能？
- **RQ2：** VLM辅助标注清洗能否改善测试CER？结果对审计提示词设计有多敏感？
- **RQ3：** 针对欠采样字符的合成数据增强在不同规模下效果如何？
- **RQ4：** 词级n-gram语言模型重评分相较Unigram beam search能带来多大提升？

## 2. 相关工作

### 2.1 面向HTR的CRNN-CTC

Shi等（2016）提出的CRNN架构将VGG风格CNN特征提取与双向LSTM序列建模结合，通过CTC损失（Graves等，2006）端到端训练，在IAM上无语言模型情况下通常达到8–12% CER。Retsinas等（2022）在残差CNN和辅助CTC shortcut加持下达到5.14%，是CRNN-CTC类方法的最佳实践水平。

当前SOTA由Transformer主导：TrOCR（Li等，2021）以预训练BEiT编码器+RoBERTa解码器在6.84亿合成行上预训练，达到2.89% CER；DRetHTR（2025）以解码器-only RetNet进一步达到2.26%。这些模型需200–600 GPU小时微调，使受控的数据层面研究难以开展，本文因此选择CRNN-CTC以保证实验可控性。

### 2.2 标注数据集中的标签噪声

标签噪声对模型泛化的危害有充分研究（Frenay & Verleysen，2014）。在NLP基准中，CoNLL-03 NER估计含5–7%标签噪声，OntoNotes4约8%，WNUT-17约18%。REVEAL（Jiang等，2025）使用多VLM协作修复图像分类测试集标签噪声。本文将类似的VLM审计理念应用于HTR领域，并发现HTR特有的挑战：手写草书字符的视觉模糊性使VLM与模型面临相同的困难，导致"噪声"与"困难但正确的标注"难以区分。

### 2.3 合成数据增强

基于字体的文本渲染（trdg，Belval 2019）是低成本扩充标注数据的常用方法，已有工作生成最多250万条合成行用于预训练（Luo等，2023）。本文的区别在于：使用LLM（豆包Seed 2.0 Pro）生成富含稀有字符的文本，针对性解决IAM分布中的类别不均衡，而非简单扩大数据量。

### 2.4 语言模型重评分

CTC beam search结合n-gram语言模型重评分（pyctcdecode + KenLM）是ASR和HTR的标准后处理方案（Hannun等，2014）。词级n-gram LM优于字符级的关键在于它能捕获词间上下文（如区分`form`与`from`），而CTC的帧级建模对此无能为力。

### 3.2 VLM标注质量审计——V1（激进提示词）

使用豆包Seed 2.0 Pro配合激进提示词（"倾向于标记任何可能的错误"）对全量10,373条样本审计，输出CORRECT / INCORRECT / AMBIGUOUS结构化判决。

| 划分 | 总数 | 被标记 | 标记率 |
|---|---|---|---|
| 训练集 | 6,482 | 2,144 | 33.1% |
| 验证集 | 976 | 360 | 36.9% |
| 测试集 | 2,915 | 1,053 | 36.1% |
| **合计** | **10,373** | **3,557** | **34.3%** |

约34%的高标记率反映了真实转录错误与VLM过度标记的混合（同义词替换、语法修正、风格改写，甚至幻觉）。

### 3.3 VLM标注质量审计——V2（保守提示词）

重新设计提示词（V2）：明确要求"默认判定为CORRECT，仅在高度确信时才标记INCORRECT"，并列举常见误报场景（数字0 vs 字母O、英式拼写、缩写、标点细微差异、草书风格变体）。

| | V1提示词 | V2提示词 |

| 训练集标记样本 | 2,144（33.1%） | 814（12.6%） |
| 保留训练样本 | 4,338 | 5,668 |

V2提示词将标记率从33.1%降至12.6%，减少标记1,330条。

## 4. 模型架构

### 4.1 CRNN-CTC

**CNN特征提取器**（VGG风格，H=64适配版）：7层卷积+BN+非对称池化，最终输出1×W/4×512。最后一层卷积核(4,1)专为H=64输入设计，实验验证优于原始(2,1)（7.35% vs 7.45%）。

**序列编码器：** 2层双向GRU，隐藏层=512，输入=512。

**输出：** Linear(1024→80)，log-softmax，80类（79字符+CTC blank）。

**损失：** CTCLoss（blank=0，zero_infinity=True，reduction='mean'）

**总参数：约1670万**

### 4.2 训练配置

| 超参数 | 取值 |
|---|---|
| 优化器 | AdamW，weight_decay=1e-4 |
| 初始学习率 | 3×10⁻⁴ |
| 学习率调度 | ReduceLROnPlateau（patience=10，factor=0.8，min_lr=1e-7） |
| 预热 | 1轮线性预热（from≈0到初始LR） |
| 批大小 | 64 |
| 梯度裁剪 | L2范数≤10 |
| Dropout | 0.1（BiGRU前） |
| 训练轮次 | 150（主实验）/ 50（超参数代理） |
| 数据增强 | 随机仿射、弹性变形、亮度抖动（仅训练集） |

## 5. 合成数据流水线

IAM训练集仅6,482条，字符分布严重不均：`#`、`*`、`;`等稀有符号出现次数不足60次。

**三层文本池：**
- 第一层：IAM训练标注原文×2（约13,000句），确保与测试分布对齐
- 第二层：LLM生成的稀有字符句子（约12,000句），每个目标字符在句中至少出现2次，目标字符包括所有数字、出现次数<500的大写字母及稀有标点
- 第三层：规则模板句子（约3,000句）作为补充

合并后约28,000句，渲染使用14种字体（9种手写风格+5种印刷体），一次性生成65,000条LMDB，实验按需取子集。

## 6. 超参数搜索

贪心单参数搜索策略：以基线配置为起点，每次改变一个参数，验证集CER改善>0.05%则保留。50轮作为代理预算。

**基线：** hidden=256，dropout=0.1，batch=64，lr=1e-4，cnn_out=512，wd=1e-4

| Run | 改变 | 验证CER | Δ | 决策 |
|---|---|---|---|---|
| 00 | 基线 | 9.04% | — | 保留 |
| **02** | **hidden=512** | **7.35%** | **−1.69%** | **保留→新最优** |
| 05 | dropout=0.3 | 7.20% | −0.15% | 保留 |
| 06 | batch=32 | 6.73% | −0.62% | 保留 |
| **08** | **lr=3×10⁻⁴** | **6.29%** | **−1.06%** | **保留→新最优** |
| 其余9组 | — | 7.20%–9.79% | 均为正 | 全部放弃 |

**最优配置：** hidden=512，lr=3×10⁻⁴，dropout=0.1，batch=64。**50轮代理CER：6.29%**

`hidden=512`带来最大提升（-1.69%），BiGRU是基线的主要瓶颈。`lr=3×10⁻⁴`带来第二大提升（-1.06%），在50轮预算内收敛更快。

## 7. 实验

### 7.1 实验设置

所有主实验使用最优配置训练150轮，以最优验证CER的checkpoint评估测试集（每组实验仅评估一次）。

评估指标：CER = edit_distance(预测, 真实) / max(len(真实), 1) × 100%

### 7.2 完整IAM vs. VLM清洗IAM（Exp1、Exp2、Exp5）

| 实验 | 训练集 | 样本数 | 测试CER | Δ vs Exp1 |
|---|---|---|---|---|
| **Exp1** — 完整IAM | `train` | 6,482 | **8.43%** | — |
| **Exp2** — V1清洗 | `train_clean` | 4,338 | **10.33%** | +1.90% ↑ 变差 |
| **Exp5-base** — V2清洗 | `train_clean_2` | 5,668 | **8.36%** | −0.07% ≈ 持平 |

**关键发现：** V1清洗（删除33%训练数据）显著恶化测试CER，从8.43%升至10.33%。这一反直觉结果表明被删除的"噪声"样本中包含大量有效训练信息。V2清洗（删除12.6%）影响可忽略（8.43%→8.36%），说明更精准的提示词可以减少误删，但仍未能使清洗效果超越完整数据集基线。

**深层原因（见7.5节）：** VLM混淆对与模型预测混淆对高度重叠，说明VLM和深度学习模型面临的是相同的视觉模糊性。VLM无法可靠区分"错误标注"与"视觉上难以分辨但正确的标注"，导致误删困难样本的副作用大于降噪收益。

### 7.5 VLM标注混淆 vs. 模型预测混淆的对比分析

这是本文最核心的分析发现。我们将两个混淆矩阵并排对比：
- **VLM混淆矩阵**：VLM审计中识别的字符替换对（IAM标注→VLM修正）
- **模型混淆矩阵**：Exp3-10x在测试集上的错误替换对（真实字符→模型预测）

**Top-10共有混淆对：**

| 字符对 | VLM标记次数 | 模型错误次数 |
|---|---|---|
| `o` ↔ `a` | 155 + 116 = **271** | 174 + 280 = **454** |
| `n` ↔ `m` | 135+... = **135+** | 105 + 89 = **194** |
| `r` ↔ `s` | 133 + 59 = **192** | 121 + 92 = **213** |
| `a` ↔ `e` | 109 + 46 = **155** | 96 + 56 = **152** |
| `n` ↔ `u` | **96+** | 58 + 102 = **160** |
| `o` ↔ `e` | **76** | 67 + 48 = **115** |
| `t` ↔ `l` | **54** | 116 + 49 = **165** |
| `h` ↔ `l` | **60** | **52** |
| `r` ↔ `n` | 56 + 41 = **97** | 76 + 66 = **142** |
| `u` ↔ `e` | **52** | **45** |

在160个两个矩阵中均出现（各自次数≥5）的字符对上，Spearman秩相关系数为 **ρ=0.559（p<0.0001）**。

**解读：** 这一显著正相关说明VLM和深度学习模型面对的是**相同的视觉困难**——草书体中`o`与`a`、`n`与`m`等字符在笔画形状上确实高度相似，这是客观的视觉模糊性，不是任何一方的系统偏差。

这一发现有两层含义：
1. **解释VLM清洗失效：** VLM之所以无法有效清洗IAM，是因为其"假阳性"（误标的混淆对）本身也是模型真实困难所在的字符对，删除这些样本反而减少了模型学习困难字符的机会。
2. **指导数据采集：** VLM混淆矩阵可作为针对性数据收集的先验——重点采集`o/a`、`n/m/u`、`r/s`等字符的清晰、无歧义手写样本，比随机扩大数据集更有效。

## 8. 语言模型解码

### 8.1 KenLM 4-gram语言模型

使用`lmplz`（Modified Kneser-Ney平滑）在183,468条句子上训练词级4-gram LM：
- IAM训练集+验证集文本标注（约7,400句）
- 20部古腾堡计划公版英文小说（约176,000行），包含《傲慢与偏见》《白鲸记》《福尔摩斯》等

最终ARPA模型150MB，词表73,540个词。

**词表大小写问题：** 初始词表构建时对所有词执行了`.lower()`，导致IAM中的专有名词（`London`、`Mr`、`Manchester`、`Evans`等大写形式）被当作未知词，在beam search中受到`unk_score_offset=-10`的惩罚。修复方式：词表同时保留原始大小写和小写形式两个版本。

### 8.2 解码结果

| 解码方式 | 模型 | 贪心CER | Beam CER | Δ vs. 贪心解码 |
|---|---|---|---|---|
| CTC贪心解码 | exp3-10x | 7.53% | — | — |
| Beam（小写Unigram词表） | exp3-10x | — | 7.50% | −0.03% |
| Beam（4-gram KenLM，小写词表） | exp3-10x | — | 6.64% | −0.89% |
| **Beam（4-gram KenLM，大小写词表）** | **exp3-10x** | — | **6.41%** | **−1.12%** |
| Beam（4-gram KenLM，大小写词表） | Exp3-1x | 7.92% | 6.58% | −1.34% |
| Beam（4-gram KenLM，大小写词表） | Exp3-3x | 7.67% | 6.61% | −1.07% |
| Beam（4-gram KenLM，大小写词表） | Exp3-5x | 7.65% | 6.56% | −1.09% |
| Beam（4-gram KenLM，大小写词表） | Exp5-5x | 7.92% | 6.90% | −1.02% |
| Beam（4-gram KenLM，大小写词表） | Exp5-10x | 8.06% | 6.87% | −1.19% |
| Beam（4-gram KenLM，大小写词表） | Exp4-3x | 8.61% | 7.37% | −1.24% |
| Beam（4-gram KenLM，大小写词表） | Exp4-10x | 9.07% | 7.64% | −1.44% |

**全局最优结果：Exp3-10x + 4-gram KenLM（混合大小写词表）= 6.41% 测试CER**

Unigram词表相较贪心解码几乎无收益（-0.03%），说明单纯的词汇约束不带概率权重时贡献有限。4-gram KenLM提供实质性改善，词汇大小写修复后额外降低0.23%，WER改善约5.6%绝对值。一个关键发现：**Exp3-1x + beam（6.58%）与Exp3-10x + beam（6.41%）仅相差0.17%**，而前者只需1/10的合成数据。这说明在beam search的加持下，超过1×的合成数据增益急剧递减——语言模型打分的效率远高于堆砌合成样本。Exp4系列即使beam后也无法接近Exp3，且Exp4-10x的贪心基线（9.07%）比Exp4-3x（8.61%）更差，证明V1清洗后过量合成数据会导致模型偏离真实分布。

## 9. 讨论

### 9.1 VLM清洗提示词设计的关键性

本文最重要的发现是VLM标注清洗对提示词设计高度敏感：V1（激进，-33%）导致CER恶化1.90%；V2（保守，-12.6%）影响可忽略。两种提示词均未能使清洗后性能超越完整数据集。

根本原因在于7.5节揭示的VLM-模型混淆对相关性：VLM和模型都在`o/a`、`n/m`、`r/s`等字符对上困难——这是手写草书的客观视觉模糊性，而不是标注错误。VLM将这些困难样本标记为"错误"，清洗后模型反而失去了学习这些困难字符的机会。

**实践建议：** 将VLM用于HTR标注审计时，应区分两类错误：（1）明显可判定的高置信度错误（如整个单词完全无关），可安全删除；（2）`o/a`、`n/m`等视觉模糊对，需结合模型自身置信度或人工验证，不应基于VLM单一判决删除。

### 9.2 合成数据的收益递减与针对性增强

Exp3曲线从0×到10×大致单调改善，但1×→3×阶段收益明显（约0.67%），5×→10×趋于平稳（约0.11%）。逐字符分析（表2）揭示：合成数据的主要贡献是修复稀有字符，而非改善常见小写字母。这验证了使用LLM生成稀有字符文本的设计价值：针对性解决分布不均衡，比均匀扩大数据量更高效。

### 9.3 语言模型与合成数据的互补性

| 方法 | CER增益（vs. Exp1基线） | 实施成本 |
|---|---|---|
| 10×合成数据增强 | 0.91%（8.43%→7.52%） | 高（数据生成+完整重训练） |
| 4-gram KenLM beam search（大小写词表） | 1.12%（7.53%→6.41%） | 低（一次性LM训练，无需重训练） |

4-gram KenLM稍优于10×合成增强，且无需重新训练模型，性价比更高。结合使用（Exp3-10x + beam）达到本文最优：**6.41% 测试CER**。

### 9.4 局限性

- **VLM清洗未能提升性能**：不代表IAM标注无误，而是说明当前VLM无法在手写字符视觉模糊性面前精准区分"真实错误"与"困难但正确的标注"
- **CRNN-CTC非SOTA架构**：与TrOCR的2.89%仍有4%差距，相同数据层面发现在Transformer架构上的效果有待验证
- **单一数据集**：仅在IAM上验证，RIMES、CVL等基准上的泛化性未知
- **合成数据领域差距**：字体渲染缺乏真实笔画变化，扩散模型生成可能带来更大收益

## 10. 结论

本文基于IAM手写数据库，以CRNN-CTC为受控设置，系统研究了超参数优化、VLM标注清洗、合成数据增强和语言模型解码对HTR性能的影响。

**主要发现：**

1. **超参数搜索**（14轮）将验证CER从9.04%降至6.29%，BiGRU隐藏层大小和学习率是最关键的两个参数。

2. **VLM清洗高度依赖提示词设计**：激进清洗（-33%）严重恶化性能；保守清洗（-12.6%）影响可忽略。VLM混淆对与模型预测混淆对之间的显著相关性（ρ=0.559）揭示了清洗失效的根本原因：两者共同面对手写草书中相同的视觉模糊性，使得"错误标注"与"困难但正确的标注"难以区分。

3. **合成数据增强**（10×）将测试CER从8.43%降至7.52%，逐字符分析确认主要收益集中于流水线针对性设计的稀有字符类别。

4. **4-gram KenLM解码**（含混合大小写词表）进一步将CER降至**6.41%**，是本文最优结果。发现IAM专有名词在小写词表中被当作未知词惩罚，修复后额外降低0.23%，是一个工程细节的可重现改进。

**未来工作：**（1）将VLM审计扩展至RIMES、CVL等HTR基准，验证混淆规律的普适性；（2）引入模型置信度对VLM标记样本进行分层清洗，仅删除模型也高置信度正确但VLM认为有误的样本；（3）基于VLM-模型共同混淆对，针对性采集`o/a`、`n/m`等字符的清晰手写样本进行数据集扩充。
